  PT server access:

    - Server has ALL corners pre-loaded on different machines/ports
    - Config: pt_server_c2dgbcptserver_cron.cfg

  Key PT commands for IFC debug:
    report_timing -through <hier_path> -max_paths 10
    report_timing -from <startpoint> -to <endpoint> -pba_mode path
    report_timing -through <signal> -path_type full_clock_expanded
    get_timing_paths -through <signal> -max_paths 1000
    get_attribute [get_cells <cell>] ref_name     # check cell type
    get_attribute [get_cells <cell>] threshold_voltage_group  # LVT/ULVT/HVT

  PBA (Path-Based Analysis):
    - More accurate than GBA (Graph-Based) — removes pessimism
    - Use -pba_mode path for single paths, -pba_mode exhaustive for full analysis
    - Typical PBA improvement: 5-15ps on setup, varies on hold
    - DCU L1 databank paths: extensive PBA analysis done in ww13_3

  Hierarchy in PT server:
    - Always use full hierarchy: icore0/par_meu/signal_name (not relative)
    - icore0 = primary core instance, icore1 = secondary (skip icore1 in analysis)
    - Port hierarchy: icore0/par_<src>/signal → icore0/par_<dst>/signal

  Cross-partition path analysis flow (from cookbook):
    Step 0.1: Identify partitions involved and crossing direction
    Step 0.2: Analyze cell types — check for LVT/HVT upgrade candidates
    Step 0.3: Find high-cap/high-delay nets (monsters)
    Step 0.4: Check logic depth (>15 stages is a red flag)
    Step 0.5: Evaluate MCP (multi-cycle path) opportunities
    Step 0.6: ULVT upgrade candidates (LVT→ULVT gain ~5-8ps per swap)
    Step 0.7: FF location sanity (distance > 500um is suspicious)
    Step 0.8: Spec budget impact (moving FF changes clock latency)

=== XML PARSING GOTCHAS ===

  CRITICAL bugs we discovered (avoid repeating):

  1. IFC parser: line.startswith('-') skips ALL negative WNS data lines!
     FIX: use line.startswith('-WNS') to skip only the header

  2. XML slack regex: slack="(-?\d+) matches normalized_slack BEFORE slack!
     Because XML attribute order is: normalized_slack="..." slack="..."
     FIX: use ' slack="(-?\d+)' with a LEADING SPACE

  3. Clock names: NOT all clocks have mclk_ prefix.
     Hold corners have: idvfreqOut, idvfreqOut_div2/4/8/16, dcmcbbclk,
     ijtag_tck, scanclk_div, scanclk_notdiv_phyex, ssh_ijtag, etc.
     Always show FULL clock names.

=== DFX (DESIGN FOR TESTABILITY) ===

  DFX timing is reported separately from functional (mclk) timing.
  DFX paths use test clocks: bist, bisr, misr, scanclk, ijtag_tck,
  bisr_ijtag, ssh_ijtag, ssnclk, slos, tap, clkg, pm_clock, saf, m4saf, async.

  DFX report structure (from Maha):
    - External paths: cross-partition DFX timing (like IFC but for test modes)
    - Internal paths: within-partition DFX timing
    - Per-domain breakdown: WNS/TNS per clock domain per partition
    - Compared TST (latest) vs REF (previous release)

  DFX report commands:
    m $MODEL/runs/core_client/n2p_htall_conf4/sta_pt/<corner>/reports/*only_dfx.x* | grep -v ifc_ | grep external | m

=== CALIBER (FC SIGNOFF) ===

  Caliber = Intel full-chip timing signoff checker (physical/electrical rules).
  Different from STA: checks cap, slopes, pulse widths, clock integrity, latch margins.
  Violations are per-pin (not per-path like STA).
  Most violations are internal to partitions (not IFC).

  Top rules: SlowSlope, MaxCap, ClockMinPulseWidth, LatchMarginWithRecovery
  Owner types: PO_PV (physical), PO_SYN (synthesis), PO (partition owner)

  Report locations:
    $MODEL/runs/core_client/n2p_htall_conf4/intel_caliber_all/<corner>/reports/caliber/
    $MODEL/runs/core_client/n2p_htall_conf4/sta_pt/reports/caliber_indicators/

  Full caliber knowledge: /nfs/site/disks/sunger_wa/skills_for_sio_copilot/caliber/
