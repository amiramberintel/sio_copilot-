================================================================================
                         COMMON REASONS FOR UNCONSTRAINED
================================================================================

1. MISSING INTERFACE SPECS
   - Partition interfaces without timing specs defined
   - Need to add specs in timing_specs/par_msid_spec.csv
   - Or add TD (top-down) constraints from core_client level

2. ASYNC INTERFACES
   - Signals crossing clock domains
   - Handled by CDC (Clock Domain Crossing) constraints
   - Not constrained for setup/hold - use set_false_path

3. FEEDTHROUGH PATHS
   - Signals that pass through without register stages
   - No internal delay to constrain
   - Constrained at source/destination partitions instead

4. DFT/SCAN SIGNALS
   - Not timing critical in functional mode
   - Constrained separately for scan mode

5. SLOW/STATIC SIGNALS
   - Control signals that change rarely
   - May use multicycle path (MCP) instead of single-cycle

6. INTENTIONALLY RELAXED
   - Some paths deemed non-critical
   - May be waived or excluded from signoff

================================================================================
                         RECOMMENDED ACTIONS
================================================================================

1. FOR MISSING SPECS:
   - Review par_msid_spec.csv for missing pins
   - Add set_input_delay / set_output_delay in SDC
   - Request specs from interface owners

2. FOR ASYNC INTERFACES:
   - Add set_false_path for true async paths
   - Add set_max_delay for quasi-static signals
   - Document in FDR exceptions

3. FOR FEEDTHROUGH:
   - Verify constrained at other partition
   - Add feedthrough_virtual_clock constraints if needed

4. CHECK COVERAGE:
   pt_shell> report_analysis_coverage -untested
   pt_shell> report_timing -from [get_ports <port>] -unconstrained
