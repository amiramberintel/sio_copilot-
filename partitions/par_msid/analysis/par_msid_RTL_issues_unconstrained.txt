FEEDTHROUGH PATHS: NO REGISTER IN PATH (14 signals)
-----------------------------------------------------------------------

PROBLEM: Signals that pass directly from input to output with no flip-flop
         These create combinational paths that can't be constrained normally

AFFECTED SIGNALS (OUTPUTS - unconstrained endpoint):
  - core_rf_bisr_chain_clk__FEEDTHRU_1  (BISR clock feedthrough)

AFFECTED SIGNALS (INPUTS - no input delay):
  - RAWhisperPsvRstrEvM203H_[0:11]__FEEDTHRU_0 (12 bits - DFX whisper)
  - dfx_sfe_vsense_ooo_vec__FEEDTHRU_0 (DFX voltage sense)

ROOT CAUSE:
  These are intentional feedthrough signals in RTL - they pass through
  the partition without registering. The naming "_FEEDTHRU_" indicates
  this was done intentionally during synthesis/place-and-route.

  This is NOT necessarily an RTL bug - may be intentional for:
  - BISR chain clock distribution
  - DFX/debug signals that need direct access
  - Voltage sensing analog signals

ACTION REQUIRED:
  1. Verify these feedthroughs are intentional (check RTL spec)
  2. Add feedthrough constraints in SDC:
     set_max_delay -from [get_ports *FEEDTHRU_0] -to [get_ports *FEEDTHRU_1]
  3. Or add virtual clock constraints for feedthrough timing

================================================================================
                      HOW TO IDENTIFY RTL ISSUES
================================================================================

1. DES-012 ERRORS - Port direction mismatch
   Command: grep "DES-012" report_error_warning.rpt
   Fix: Compare RTL port list with spec file

2. UNDRIVEN OUTPUTS - Output ports not driven by any logic
   Command: pt_shell> report_port -unconnected
   Fix: Check RTL for missing assignments

3. UNLOADED INPUTS - Input ports not used by any logic  
   Command: pt_shell> report_port -unconnected
   Fix: Check RTL for unused inputs (may be intentional)

4. FEEDTHROUGH - No register in path
   Look for: *_FEEDTHRU_* in port names
   Fix: Verify intentional or add missing registers

5. FLOATING NETS - Nets with no driver
   Command: pt_shell> all_fanin -flat -to [all_registers] -trace_arcs all
   Fix: RTL assignment missing
