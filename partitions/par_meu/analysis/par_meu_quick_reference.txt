COMMON PT CLIENT COMMANDS
--------------------------
# Report timing to a MEU output port (setup):
pt_client.pl -corner func.max_high.T_85.typical \
  "report_timing -to icore0/par_meu/mowbcheckrobidldm803h_*_[*] -max_paths 5"

# Report timing from a MEU input port (setup):
pt_client.pl -corner func.max_high.T_85.typical \
  "report_timing -from icore0/par_meu/rslduopcodm301h_*_[*] -max_paths 5"

# Check clock latency at MEU endpoint:
pt_client.pl -corner func.max_high.T_85.typical \
  "report_clock_timing -type latency -to [get_pins icore0/par_meu/*/CP]"

# Get all failing paths in MEU:
pt_client.pl -corner func.max_high.T_85.typical \
  "report_timing -through icore0/par_meu/* -slack_lesser_than 0 -max_paths 50"

KEY TERMINOLOGY
---------------
  WNS     Worst Negative Slack (most failing path, in ps)
  TNS     Total Negative Slack (sum of all failing paths)
  FEP     Failing End Points (count of failing endpoints)
  LOS     Level Of Success (target slack for a given model)
  CI      Clock Integration (clock tree synthesis/tuning)
  ICG     Integrated Clock Gate (clock gating cell)
  TIP     Topology In-context Plan (.tp file for routing)
  SIO     Spec I/O (interface spec between partitions)
  PO      Port Owner (who owns the constraint/spec)
  SIO2PO  Reassignment of port from one TIP cell to another
  PRF     Physical Register File (large macro in MEU)
  MOB     Memory Order Buffer
  SDB     Store Data Buffer
  ROB     Re-Order Buffer
  RS      Reservation Station
  FT      Feed-Through (signal passes through without terminating)
  PBA     Path-Based Analysis (more accurate than GBA)
  GBA     Graph-Based Analysis (default, pessimistic)
  OCV     On-Chip Variation (derating)
