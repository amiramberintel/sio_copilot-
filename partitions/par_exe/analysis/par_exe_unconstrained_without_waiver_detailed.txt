1. rsveceuuopcodm303h_* (112 endpoints) - VECTOR EU MICRO-OP CODE
-------------------------------------------------------------------------------

SIGNALS: rsveceuuopcodm303h_0_[0:13] through rsveceuuopcodm303h_7_[0:13]
         8 instances x 14 bits each = 112 signals

FUNCTION:
  - RS (Reservation Station) output to Vector Execution Unit
  - Carries micro-op code for vector operations
  - Controls what vector operation the EU will perform
  - Stage M303H indicates pipeline stage 303 in high phase

INTERFACE: par_exe → par_ooo_vec (Out-of-Order Vector unit)

ROOT CAUSE:
  - Missing set_output_delay constraint in par_exe_spec.csv
  - Interface timing specification not defined between par_exe and par_ooo_vec
  - These outputs need timing budget allocation for the receiving partition

IMPACT IF NOT FIXED:
  - Timing path from par_exe flops to par_ooo_vec flops is not verified
  - Could cause setup violations at chip integration
  - May result in functional failures in vector operations

ACTION REQUIRED:
  Option A: Add timing spec (RECOMMENDED)
    - Add to par_exe_spec.csv:
      rsveceuuopcodm303h_*,output,mclk_exe,0.100,setup
    - Or add SDC constraint:
      set_output_delay -clock mclk_exe -max 0.100 [get_ports rsveceuuopcodm303h_*]
  
  Option B: Add waiver if interface is async or MCP
    - Add to unconstrained.waivers.csv:
      par_exe/rsveceuuopcodm303h_*,<owner_name> - <justification>

OWNER: Vector Execution Unit PO (need to confirm with OOO_VEC team)

3. specpdstforldbundlecancelm801h_* (32 endpoints) - SPEC PDST FOR LD BUNDLE CANCEL
-------------------------------------------------------------------------------

SIGNALS: specpdstforldbundlecancelm801h_0_[0:3] through _7_[0:3]
         8 instances x 4 bits each = 32 signals

FUNCTION:
  - Speculative Physical Destination for Load Bundle Cancel
  - Used in memory disambiguation and load speculation
  - When a load is cancelled, this signal indicates which physical 
    destination register was speculatively allocated
  - Stage M801H = late pipeline stage (stage 801)

INTERFACE: par_exe → par_ooo_int or par_meu (Memory Execution Unit)

ROOT CAUSE:
  - Missing output delay spec for load cancellation interface
  - This is a late-stage signal (801) so timing may be critical

IMPACT IF NOT FIXED:
  - Load speculation/cancellation timing not verified
  - Could cause incorrect register allocation on mis-speculation

ACTION REQUIRED:
  - Add timing spec with appropriate budget for late-stage signal
  - May need aggressive timing (stage 801 is late in pipe)
  - Example: set_output_delay -clock mclk_exe -max 0.050 [get_ports specpdstforldbundlecancelm801h_*]

OWNER: Load/Store Unit PO or MEU PO
