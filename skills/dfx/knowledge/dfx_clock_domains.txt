NOTE: DFX clocks are much SLOWER than mclk (186ps).
  - scanclk at 3704ps is 20x slower than mclk
  - JTAG at 6266ps is 34x slower than mclk
  - Yet DFX paths still fail because the logic chains are very long
    (scan chains, JTAG shift registers, BISR mux chains)

DOMAIN NAMES IN DFX REPORTS:
  These appear in Maha's domain breakdown tables:
  - mclk        = functional clock paths tested in DFX mode
  - lcp         = low-cost patterns (structural test)
  - bist        = BIST engine clock domain
  - bisr        = BISR mux chain domain
  - misr        = MISR (Multiple-Input Signature Register)
  - slos        = SLOS (Scan-Logic-On-Scan) test patterns
  - clkg        = clock gating test domain
  - pm_clock    = power management clock domain
  - tap         = TAP (Test Access Port) domain
  - async       = asynchronous crossings in DFX
  - m4saf       = mclk4saf stuck-at-fault domain
  - saf         = stuck-at-fault domain
  - ijtag_tck   = IJTAG clock domain
  - bisr_ijtag  = BISR accessed via IJTAG
  - ssh_ijtag   = Secure Scan Host IJTAG
  - ssnclk      = SSN test clock domain
  - scanclk     = scan chain shift domain
  - ssnclk      = simultaneous switching noise
