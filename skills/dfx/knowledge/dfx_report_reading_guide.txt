READING THE BUCKET TABLE:
  model  par       ext_0ps  ext_5ps  ext_10ps  ext_20ps  ...  ext_wns  ext_tns
  TST    par_meu   17       15       2         0              -13      -129

  - ext_0ps = total failing paths (WNS < 0)
  - ext_5ps = paths with WNS < -5ps (subset of 0ps)
  - ext_10ps = paths with WNS < -10ps (subset of 5ps)
  - Buckets are CUMULATIVE: 0ps >= 5ps >= 10ps >= ...
  - ext_wns = worst negative slack (most negative = worst path)
  - ext_tns = total negative slack (sum of all failing paths)
  - Blank = no failing paths (clean)

  External = paths crossing partition boundaries
  Internal = paths within the partition

READING THE DOMAIN BREAKDOWN:
  partition  WNS/TNS  mclk  bist  bisr  ...
  par_meu    WNS      -13   -8    -8
  par_meu    TNS      -360  -160  -160

  - Each column is a DFX clock domain
  - WNS row = worst path in that domain for that partition
  - TNS row = sum of all failing paths in that domain
  - Blank = no failing paths in that domain
  - A path can appear in multiple domains (cross-domain paths)

COMPARING TST vs REF:
  - If TST < REF paths: IMPROVEMENT (fewer violations)
  - If TST > REF paths: REGRESSION (more violations)
  - Watch for:
    * New synthesis bringing new paths (par_fmav0: 0 -> 3104)
    * Wrong DFT collaterals tag (par_ooo_int regression)
    * New CIs changing RTL (par_pmh: 1108 -> 1 = massive improvement)
