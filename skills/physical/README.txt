GOTCHAS - EXTRACTION
---------------------
  - Shorts > 0 does NOT always block signoff -- check with PO
    if shorts are on non-critical nets
  - Opens affect SPEF accuracy -- check if open nets are on timing-critical paths
  - smin violations = manufacturing rule issues -- escalate to PO
  - Compare shorts trend across builds -- increasing = routing degradation
  - CKSUM lines verify SPEF integrity -- if cksum mismatch, extraction is stale

  PART 2: DUET / CALIBER DRC

WHAT IS DUET
------------
Duet is Intel's timing DRC framework running on top of Caliber.
It checks physical+timing quality rules on each partition after CTS/route.
Results are in the APR summary report.

APR SUMMARY LOCATION
---------------------
  <ward>/runs/<par>/n2p_htall_conf4/intel_caliber/<scenario>/reports/<par>.apr.summary

Scenarios run per partition:
  - func.max_low.T_85.typical      (main setup check)
  - func.max_med.T_85.typical
  - func.max_high.T_85.typical
  - func.max_nom.T_85.typical
  - func.min_low.T_85.typical      (hold check)
  - fresh.min_fast.F_125.rcworst_CCworst

APR SUMMARY FORMAT
------------------
Section 1: Runtime Info
  - Duet version, ruleset paths, stage runtimes

Section 2: Queries Data (per-rule table)
  Columns:
  TV  = Total Violations
  CV  = Compressed Violations
  GW  = Globally Waived
  UW  = User Waived
  RW  = Redundant Waivers
  CW  = Collateral Waived
  UA  = User Approved
  RA  = Redundant Approvals
  CA  = Collateral Approved
  RV  = Remaining Violations (NOT waived -- THIS IS THE KEY NUMBER)
  Severity = fatal / must / review / recommended / info
  Run status = OK / FAIL

Section 3: Quality Summary
  Total number of rules with errors
  Total number of violations
  Total number of globally waived
  Total number of remaining violations          <-- KEY METRIC
  Total number of remaining violations (fatal)  <-- BLOCKS SIGNOFF
  Total number of remaining violations (must)
  Total number of remaining violations (review)

INDIVIDUAL CALIBER REPORT FORMAT
---------------------------------
Each rule produces a .rpt in:
  <ward>/runs/<par>/.../intel_caliber/<scenario>/reports/caliber/<RuleName>.rpt

Format: TSV (tab-separated), first line = column headers, then violations.
Last 3 lines = TIMESTAMP, HOSTNAME, USER.
Empty body (headers only) = no violations (clean).

QUICK COMMANDS - DUET DRC
--------------------------
Check total remaining violations:
  grep "remaining violations" \
    <ward>/runs/<par>/.../intel_caliber/func.max_low.T_85.typical/reports/<par>.apr.summary

Check which rules have remaining violations (non-zero RV column):
  grep -v "| 0 " <par>.apr.summary | grep "|"

Check missing extraction data (common SIO issue):
  cat <ward>/runs/<par>/.../reports/caliber/DataNetMissingExtractionData.rpt
  cat <ward>/runs/<par>/.../reports/caliber/ClockNetMissingExtractionData.rpt

Compare DRC across builds:
  for ward in <ward1> <ward2>; do
    echo "=== $ward ==="
    grep "remaining violations" \
      $ward/runs/<par>/.../intel_caliber/func.max_low.T_85.typical/reports/<par>.apr.summary
  done

Check all partitions DRC in one build:
  for par in par_meu par_exe par_fe par_mlc par_msid par_ooo_int \
             par_ooo_vec par_pm par_pmh; do
    echo "--- $par ---"
    grep "remaining violations" \
      <ward>/runs/$par/n2p_htall_conf4/intel_caliber/func.max_low.T_85.typical/reports/$par.apr.summary
  done

CALIBER WAIVERS
----------------
Location: <ward>/runs/<par>/n2p_htall_conf4/caliber_waiver/
Contains:
  collateral/         -- Project-level waivers (auto-applied)
  <par>_waiver_db/    -- User waivers for this partition
  <par>_approval_db/  -- Approved violations

Waivers reduce TV to GW count. Only RV (remaining) matters for signoff.

SCRIPTS
  duet_summery.csh       -- Get violation count for one rule+partition
                            Usage: duet_summery.csh <fub> <rule>
                            Requires: $FUB_PV_ARCHIVE
  duet_core_summery.csh  -- Get violation count across ALL partitions for a rule
                            Usage: duet_core_summery.csh <rule>
                            Requires: $FCT_MODEL, $PROJECT
  DuetruleStatistic      -- Cross-partition DRC statistics
                            Shows: violating_fubs, avg_per_fub, total_violations
                            Requires: $FUB_PV_ARCHIVE

GOTCHAS - DRC
--------------
  - "remaining violations (fatal)" > 0 = BLOCKS SIGNOFF
  - "remaining violations (must)" > 0 = needs waiver or fix before signoff
  - ClockNetMissingExtractionData = usually re-extraction needed
  - DataNetMissingExtractionData = common after ECO, re-extract to fix
  - NoFuncClocksOnRegisters high GW count (24K+) is normal -- globally waived
  - MinClockPeriod violations = cells violating min pulse width -- may need
    clock freq reduction or cell swap
  - Always check the SPECIFIC scenario -- func.max_low is the primary one

================================================================================
  PART 4: CROSS-BUILD PHYSICAL HEALTH
================================================================================

No automated script yet (GAP). Recommended workflow:
  1. For each build, collect:
     - extract_quality shorts/opens/smin counts per partition
     - APR summary remaining violations per partition
  2. Compare across daily/release builds
  3. Flag regressions: shorts increasing, new DRC violations
  4. Track in daily monitoring alongside timing metrics

Trend indicators:
  - Shorts increasing = routing congestion or ECO side effects
  - Missing extraction data increasing = extraction not re-run after ECO
  - MinClockPeriod violations = clock tree needs work
  - New "must" violations = investigate immediately
