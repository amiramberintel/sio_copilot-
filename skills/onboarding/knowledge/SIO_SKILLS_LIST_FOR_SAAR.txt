  SKILL #2: CROSS-PARTITION TIMING DEBUG (IFC)
================================================================================

  What: Debug inter-partition (IFC) timing failures — the core SIO task.
        Trace failing paths across partition boundaries, identify root cause,
        propose fixes.

  Who needs it: Every SIO — this is the core competency.

  Cookbooks:
    cookbooks/cross_partition_debug_playbook.txt
    cookbooks/cross_partition_debug_with_pt_client_cookbook.txt + .sh
    cookbooks/cross_partition_debug_with_pt_client_README.txt
    cookbooks/master_timing_fix_cookbook.txt + README
    cookbooks/compare_tst_ref_cookbook.txt

  Knowledge:
    knowledge/cross_partition_debug_playbook.txt    — Master debug flow
    knowledge/cross_partition_timing_fix_guideline.txt — Fix principles
    knowledge/path_analysis_procedure.txt           — Step-by-step path debug
    knowledge/SPEF_explanation.txt                  — SPEF parasitic format
    knowledge/FC_VCLP_explanation.txt               — VCLP explanation

  Debug flow (8 steps from bootstrap):
    Step 0.1: Identify partitions involved and crossing direction
    Step 0.2: Analyze cell types — LVT/HVT upgrade candidates
    Step 0.3: Find high-cap/high-delay nets (monsters)
    Step 0.4: Check logic depth (>15 stages = red flag)
    Step 0.5: Evaluate MCP (multi-cycle path) opportunities
    Step 0.6: ULVT upgrade candidates (LVT->ULVT gain ~5-8ps per swap)
    Step 0.7: FF location sanity (distance > 500um = suspicious)
    Step 0.8: Spec budget impact (moving FF changes clock latency)

================================================================================

        signal routing — repeater insertion, buffer spacing, layer assignment.

  Who needs it: SIOs working on cross-partition routing optimization.

  Files (tip/):

  Key parameters:
    Buffer spacing: 170-200um typical for N2P
    Metal layers: M17/M18 for long-distance signals
    Repeater count = ceil(distance / buffer_spacing)

    One file per PVT corner, pattern:
      <mode>.<timing>.<corner>.<rc>__core_client_io_constraints.tcl

  Corner naming: <mode>.<voltage_corner>.<process_temp>.<rc_corner>
    Mode:    func (aged BTI), fresh (no aging)
    Process: T=typical, F=fast, S=slow
    Temp:    85=85C, 125=125C, M40=-40C
    RC:      typical, cworst, rcworst, CCworst, CCworst_T

  Key voltages (VCCCORE):
    max_low=0.550V  max_nom=0.650V  max_med=0.850V  max_high=1.100V
    max_turbo=1.210V  min_fast=1.271V  min_slow=0.540V
