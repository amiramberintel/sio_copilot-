WHAT CAN BE AUTO-CONVERTED (mechanical sed/awk):
════════════════════════════════════════════════════════════════════════════

  ✅ Header comments (NLIB, BUILD_DIR, etc.)
  ✅ Netlist check section (nets/pins/partitions — IDENTICAL, same RTL)
  ✅ Buffer cell name in NER
  ✅ Metal layer names in NER, shape_layers, layer_cutting_distance
  ✅ NDR rule name replacement (if we have the mapping)
  ✅ Routing rule definitions (swap GFC metal widths → PNC metal widths)

════════════════════════════════════════════════════════════════════════════
WHAT CANNOT BE AUTO-CONVERTED (needs JNC-specific data):
════════════════════════════════════════════════════════════════════════════

  ⚠️  Buffer spacing — varies per signal, might differ for JNC
  ⚠️  bbox_percentage — completely different floorplan
  ⚠️  Origin coordinates — different die size
  ⚠️  Repeater offsets — different wire lengths
  ⚠️  Number of repeaters per edge — depends on wire length/buffer spacing
  ⚠️  Repeater height/width multipliers (1.5/1.5 vs 2.0/1.2)
  ⚠️  NDR rules without clear 1:1 mapping
  ⚠️  Shape_layer percentages (routing path differs)
  ⚠️  Shape_layer widths (GFC has 22.68, PNC has 0.96)
