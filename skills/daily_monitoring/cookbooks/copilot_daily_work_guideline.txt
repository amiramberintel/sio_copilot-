  4. TOOL REFERENCE — SCRIPTS
  ════════════════════════════════════════════════════════════════════════════════

  All scripts in: /nfs/site/disks/sunger_wa/sio_scripts/
  All are executable (chmod +x, shebang #!/usr/bin/env python3)

  ─── run_daily_update.py ──────────────────────────────────────────────────────

  PURPOSE: Automate the entire daily update flow (Steps 0-7 of cookbook)

  USAGE:
    ./run_daily_update.py                      # auto-detect latest daily
    ./run_daily_update.py --tag ww13_2         # custom output dir name
    ./run_daily_update.py --dry-run            # preview without executing

  WHAT IT DOES:
    1. Finds the latest daily build automatically
    2. Creates output directory (ww<WW>_<day>/)
    3. Copies par_status.py from daily, patches it
    4. Generates par_*_status.txt for all 11 partitions
    5. Generates diff vs previous output
    6. Runs check_daily_vs_po.py for version comparison
    7. Prints executive summary

  OUTPUT FILES:
    ww<WW>_<day>/par_meu_status.txt
    ww<WW>_<day>/par_ooo_int_status.txt
    ...  (11 files)
    ww<WW>_<day>/diff_report.txt
    ww<WW>_<day>/version_check.txt

  README: sio_scripts/README_run_daily_update.txt

  ─── check_daily_vs_po.py ────────────────────────────────────────────────────

  PURPOSE: Compare daily build vs PO (partition owner) workspace versions

  USAGE:
    ./check_daily_vs_po.py                     # auto-detect latest daily
    ./check_daily_vs_po.py --daily /path/to/daily
    ./check_daily_vs_po.py -o report.txt       # save to file

  WHAT IT SHOWS:
    Per partition: daily VER_NNN vs PO LATEST VER_NNN
    Status: MATCH / DAILY_AHEAD / DAILY_BEHIND

  WHY IT MATTERS:
    If daily is behind PO → fixes exist but aren't in the daily yet
    If daily matches PO → what you see is what PO delivered

  README: sio_scripts/README_check_daily_vs_po.txt

  ─── check_spec_vs_constraints.py ────────────────────────────────────────────

  PURPOSE: Find gaps between SIO spec budgets and io_constraints.tcl

  USAGE:
    ./check_spec_vs_constraints.py --par par_meu       # single partition
    ./check_spec_vs_constraints.py --par all            # all partitions summary
    ./check_spec_vs_constraints.py --par par_meu --use-po  # check PO workspace
    ./check_spec_vs_constraints.py --par par_meu -o report.txt

  WHAT IT CHECKS:
    Spec CSV (par_*_spec.csv from arc LATEST):
      - Contains port names, spec budgets (ps), clock, direction
    io_constraints.tcl (from daily FCL or PO):
      - Contains set_load, set_output_delay, set_input_delay
    IFC report (par_*_ifc.rpt from daily):
      - Contains WNS per port from chip-level STA

  OUTPUT CATEGORIES:
    ✓ Spec + constraint exist   → working correctly
    ✗ Spec, NO delay            → REAL GAP (PO can't optimize this path!)
    ? Spec, port not in model   → bus mismatch or new RTL
    FAILING & UNCONSTRAINED     → gap ports with negative WNS in IFC

  HOW THE FLOW WORKS:
    par_*_spec.csv ──▶ FCT model gen ──▶ io_constraints.tcl ──▶ PO STA (PT)
    (spec budget)      (by ayarokh)     (set_output_delay)     (times path)

    set_load ALONE = PT knows cap, but has NO timing target
    set_output_delay = PT creates budget → reports violations → optimizes

  COMMON FINDING:
    io_constraints.tcl not regenerated after spec CSV update
    → new specs exist but PO never sees them → ports are UNCONSTRAINED

  README: sio_scripts/README_check_spec_vs_constraints.txt

  6. TOOL REFERENCE — PT_CLIENT (live timing queries)
  ════════════════════════════════════════════════════════════════════════════════

  The pt_client.pl tool sends PrimeTime commands to a live server.
  Results in 5-10 seconds (vs 2+ hours to open PT GUI).

  SETUP:
    source /nfs/site/proj/lnc/c2dgbcptserver_sc8_fct/cth2_ptserver/cth2_ptserver_root/aliases_for_pt_client

  ALIASES (after setup):
    pt  '<command>'     → worst setup corner (func.max_high.T_85.typical)
    pth '<command>'     → worst hold corner  (func.min_low.T_85.typical)

  DIRECT USAGE:
    sg soc -c "/nfs/site/proj/lnc/c2dgbcptserver_sc8_fct/cth2_ptserver/cth2_ptserver_root/pt_client.pl \
      -m latest_gfcn2clienta0_bu_prp_func.max_high.T_85.typical \
      -c '<PT_COMMAND>'"

  COMMON QUERIES:

    Report timing through a port:
      pt 'report_timing -through [get_pins icore0/par_meu/<portname>] -max_paths 3'

    Report timing from FF to FF:
      pt 'report_timing -from [get_pins icore0/par_meu/<startFF>/CP] \
          -to [get_pins icore0/par_fmav1/<endFF>/D] -max_paths 1'

    Check if port is constrained:
      pt 'report_port [get_ports icore0/par_meu/<portname>]'

    Get clock latency:
      pt 'report_clock_timing -type latency [get_clocks mclk_meu]'

    Check path groups:
      pt 'report_timing -path_group <group> -max_paths 5'

  IMPORTANT:
    • Always use icore0/ prefix for cell/pin paths
    • Use get_pins for internal pins, get_ports for top-level ports
    • Escape brackets: \[0\] or use {portname[0]}
    • Read-only tool — cannot apply ECOs

  COOKBOOK: cross_partition_debug_with_pt_client_cookbook.txt
  TOOL REF: pt_client_cookbook.txt
